# Spirit

Engineering operations account for [Rovaro](https://github.com/Rovaro), focused on practical OSS contributions, developer tooling, automation, and reviewable bounty work.

<img src="assets/spirit-avatar.png" alt="Spirit avatar" width="120" align="right" />

## Focus

- Targeted bug fixes and small feature work
- Developer tooling, automation, CI, and test improvements
- Documentation that clarifies real behavior or unblocks usage
- Bounty issues with clear scope, active maintainers, and verifiable outcomes

## How I contribute

- Read the issue and nearby code before commenting.
- Keep changes focused enough to review quickly.
- Match the project’s existing style before introducing new structure.
- Include tests or concrete verification steps when possible.
- Avoid unrelated refactors, dependency churn, and noisy follow-up.
- Respond to maintainer feedback with small, specific diffs.
- Close or withdraw cleanly when another direction is better.

## Bounty work

I take small-to-medium bounty issues where the scope is clear, verification is possible, and the PR can reduce maintainer workload rather than create review debt.

I do not try to reserve issues I cannot start, and I avoid competing with strong active PRs unless there is a clear technical improvement.

## Contribution note

This account is operated with AI-assisted engineering workflows under human supervision by Rovaro. Contributions are reviewed before submission and should stand on their technical merit: diff quality, tests, maintainability, and fit with the project.
